Multi-Tier Architecture

Not One Model. Intelligent Routing.

The orchestrator chooses the right tier by user intent, device capability, queue pressure, and cost policy.

Tier 0

On-Device (Mobile)

Use DM-VTON / Mobile-VTON for privacy-first instant previews. Best for mobile apps and poor networks.

Tier 1

Fast Cloud Preview

FastFit / CatVTON / FASHN-VTON for 1–3s response. This is the default path for most web sessions.

Tier 2

Premium Quality

IDM-VTON / OOTDiffusion / DualFit / DS-VTON as async jobs for final export-level outputs.

Video

Live / Video Try-On

Lucy 2.0 / MagicTryOn-style path for high-fidelity temporal rendering and live camera experiences.

Orchestrator Brain

Routing Rules That Keep UX Fast

Decision Inputs

  • User intent: preview vs final export vs live video
  • Device profile: mobile power, browser support, connection quality
  • Queue health: GPU utilization and current p95 wait
  • Cache hit state: person/garment hash reuse
  • Budget policy: per-session inference cost guardrails

Routing Outputs

  • Immediate Tier-1 preview response
  • Parallel Tier-2 async enqueue (optional)
  • Fallback to reduced resolution under load
  • Video path isolation from image queue starvation
  • Graceful degradation instead of user-visible failure
Execution Roadmap

Build Plan in 6 Phases

A practical order that validates value early and postpones expensive complexity until traction is proven.

Phase 1 — Foundation Weeks 1-3

  • Ingestion API, storage, preprocessing service, baseline evaluation set.
  • Pose/parsing + canonical image normalization.

Phase 2 — Tier 1 Launch Weeks 4-6

  • Fast cloud model + cache + web UI flow.
  • Target: stable 1–3 second preview path.

Phase 3 — Orchestrator + Queue Weeks 7-9

  • Routing engine, async workers, retry/fallback policies.
  • Progressive delivery (preview first, HQ later).

Phase 4 — Tier 2 Quality Weeks 10-12

  • Integrate HQ diffusion path and benchmark quality/cost tradeoffs.
  • Download-ready outputs and signed links.

Phase 5 — Video Path Weeks 13-15

  • Add live/video try-on service with temporal consistency constraints.
  • Isolate video GPU pools from image workloads.

Phase 6 — Scale & Market Fit Weeks 16+

  • Uzbek catalog integrations, analytics loops, AB tests, monetization modes.
  • Fine-tune data strategy and enterprise API packaging.

Risk Register

Known Risks + Mitigation

Latency Spikes

Mitigate with queue backpressure, tier fallback, regional inference pools, and cache-first returns.

Model Drift / Artifacts

Keep golden test set, visual diff checks, periodic offline eval, and canary deploy strategy.

Cost Overrun

Set per-route budget policy; use progressive delivery to avoid sending every request to expensive tier paths.

Privacy & Compliance

Auto-delete policy, signed URLs, encryption-at-rest, audit logs, and optional on-prem enterprise mode.

Licensing Constraints

Track model-level commercial rights and maintain swappable model adapters in architecture.

Vendor Dependency

Avoid lock-in with standardized inference contracts and dual provider strategy for video endpoints.
