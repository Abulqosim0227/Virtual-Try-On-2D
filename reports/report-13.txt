HONEST ASSESSMENT
=================
CatVTON's demo images on GitHub are cherry-picked results with their exact
detectron2 DensePose (which we can't install on our stack). Our TorchScript
DensePose produces similar but not identical segmentation. The mask quality
gap between our TorchScript version and their original detectron2 version
may explain the remaining artifacts.

Options going forward:
1. Accept current quality (good for some poses, bad for others) and build the web UI
2. Try a completely different model (FASHN-VTON, Re-CatVTON, etc.)
3. Set up detectron2 in a Docker container for one-time DensePose mask generation
